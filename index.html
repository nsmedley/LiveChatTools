Moneypenny.openProactiveChat(forced, offlineFallback, msg);

Forced - Should Moneypenny disregard whether or not the visitor has recently closed a chat window? And should Moneypenny disregard whether or not you have “Enable Proactive Chat” checked in your Configurator? (this needs to be set to either True or False)

offlineFallback - Should this function fallback to the Offline form if live chat is currently unavailable? (this needs to be set to either True or False)

msg - When calling .openProactiveChat() manually, you must manually set a first-message for the Proactive Chat. This message will appear to come from your agent, and will show up wherever you call this manually. So in most cases, the best option is to define a very generic message such as, ‘Thanks for visiting. How can I help you?’

An example of this in action would be:

Moneypenny.openProactiveChat(true, true, “This is a custom proactive chat”);

To set a delay all they need to do is wrap this inside of a setTimeout. Such as the below:

setTimeout(function(){
    Moneypenny.openProactiveChat(true, true, “This is a custom proactive chat”);
}, 1000);
